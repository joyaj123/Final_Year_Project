LOG IN :
A token is like a digital ID card the server gives you after you log in.
1️⃣ User sends email + password to the server
2️⃣ The controller receives the request
3️⃣ The server searches the database for that email
4️⃣ It compares the password with bcrypt
5️⃣ If correct → create a JWT token
6️⃣ Send the token + user info back



Every time i create a controller i should associated to it a route than conect that route to server.js. 
Each controller has a route, route is a way to talk with the backend
